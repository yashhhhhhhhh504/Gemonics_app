# NGS Analysis Platform

A full-stack web application for automated next-generation sequencing (NGS) data analysis. Upload raw sequencing data, run validated genomics pipelines through a simple interface, and obtain clear, interpretable results — no command-line expertise required.

## Features

- **User Management** — Register, login, role-based access
- **Project Dashboard** — Organize samples by project with metadata
- **File Upload** — Upload FASTQ, BAM, or VCF files via the web interface
- **Pipeline Selection** — WGS, WES, Gene Panel, and RNA-Seq workflows
- **Run Modes** — Standard, Fast (low-resource), and High Sensitivity
- **Real-time Monitoring** — Track job progress, view live logs
- **Resource Management** — CPU/RAM/disk monitoring, configurable limits
- **Results & Reports** — HTML reports, downloadable VCFs, BAMs, and QC summaries
- **Reproducibility** — All parameters and versions logged per run
- **Tool Status Check** — Built-in page showing installed tools with download links

## Quick Start

### Option 1: Docker (Recommended)

```bash
git clone https://github.com/YOUR_USERNAME/ngs-analysis-platform.git
cd ngs-analysis-platform
chmod +x setup.sh
./setup.sh
```

Then open **http://localhost:8000** in your browser.

### Option 2: Local Installation

```bash
git clone https://github.com/YOUR_USERNAME/ngs-analysis-platform.git
cd ngs-analysis-platform
chmod +x setup-local.sh
./setup-local.sh

# Start the server
source venv/bin/activate
cd backend
python main.py
```

Then open **http://localhost:8000** in your browser.

## Bioinformatics Tools

The platform integrates the following tools. When using Docker, all tools are pre-installed. For local setup, install the tools you need:

### Core Tools (Required)

| Tool | Purpose | Install |
|------|---------|---------|
| **SAMtools** 1.19+ | BAM/SAM manipulation | [htslib.org/download](https://www.htslib.org/download/) |
| **BWA** 0.7.18+ | Short read alignment | [github.com/lh3/bwa](https://github.com/lh3/bwa) |
| **bcftools** 1.19+ | Variant calling & VCF manipulation | [htslib.org/download](https://www.htslib.org/download/) |

### QC & Preprocessing

| Tool | Purpose | Install |
|------|---------|---------|
| **FastQC** 0.12+ | Read quality assessment | [babraham.ac.uk/projects/fastqc](https://www.bioinformatics.babraham.ac.uk/projects/fastqc/) |
| **fastp** 0.23+ | Adapter trimming & quality filtering | [github.com/OpenGene/fastp](https://github.com/OpenGene/fastp) |
| **MultiQC** 1.20+ | Aggregate QC reports | `pip install multiqc` or [multiqc.info](https://multiqc.info/) |

### Variant Calling & Annotation

| Tool | Purpose | Install |
|------|---------|---------|
| **GATK** 4.5+ | HaplotypeCaller, BQSR, variant filtering | [github.com/broadinstitute/gatk](https://github.com/broadinstitute/gatk/releases) |
| **SnpEff** 5.2+ | Functional variant annotation | [pcingola.github.io/SnpEff](https://pcingola.github.io/SnpEff/) |

### RNA-Seq Tools

| Tool | Purpose | Install |
|------|---------|---------|
| **HISAT2** 2.2+ | Splice-aware RNA-seq alignment | [daehwankimlab.github.io/hisat2](http://daehwankimlab.github.io/hisat2/) |
| **minimap2** 2.28+ | Long read / RNA-seq alignment | [github.com/lh3/minimap2](https://github.com/lh3/minimap2) |

## Reference Genomes

Place reference files in the `data/references/` directory:

```
data/references/
├── genome.fa           # Reference FASTA (required)
├── genome.fa.fai       # FASTA index (auto-generated by samtools)
├── genome.dict         # Sequence dictionary (auto-generated by GATK)
├── known_sites.vcf.gz  # Known variant sites for BQSR (optional)
└── genes.gtf           # Gene annotation for RNA-seq (optional)
```

### Download GRCh38 (Human)

```bash
# Reference genome
wget https://ftp.ncbi.nlm.nih.gov/genomes/all/GCA/000/001/405/GCA_000001405.15_GRCh38/seqs_for_alignment_pipelines.ucsc_ids/GCA_000001405.15_GRCh38_no_alt_analysis_set.fna.gz
gunzip GCA_000001405.15_GRCh38_no_alt_analysis_set.fna.gz
mv GCA_000001405.15_GRCh38_no_alt_analysis_set.fna data/references/genome.fa

# Index the reference
samtools faidx data/references/genome.fa
bwa index data/references/genome.fa

# Known sites for BQSR (optional)
wget https://storage.googleapis.com/genomics-public-data/resources/broad/hg38/v0/Homo_sapiens_assembly38.dbsnp138.vcf.gz
wget https://storage.googleapis.com/genomics-public-data/resources/broad/hg38/v0/Homo_sapiens_assembly38.dbsnp138.vcf.gz.tbi
mv Homo_sapiens_assembly38.dbsnp138.vcf.gz data/references/known_sites.vcf.gz
mv Homo_sapiens_assembly38.dbsnp138.vcf.gz.tbi data/references/known_sites.vcf.gz.tbi

# Gene annotation for RNA-seq (optional)
wget https://ftp.ensembl.org/pub/release-110/gtf/homo_sapiens/Homo_sapiens.GRCh38.110.gtf.gz
gunzip Homo_sapiens.GRCh38.110.gtf.gz
mv Homo_sapiens.GRCh38.110.gtf data/references/genes.gtf
```

### Download GRCm39 (Mouse)

```bash
wget https://ftp.ensembl.org/pub/release-110/fasta/mus_musculus/dna/Mus_musculus.GRCm39.dna.primary_assembly.fa.gz
```

## Usage Guide

### 1. Register & Login

Open the platform in your browser and create an account.

### 2. Create a Project

Click **"+ New Project"** on the dashboard. Set the organism, genome build, and description.

### 3. Upload Samples

Inside your project, click **"Upload Samples"**. Upload paired-end FASTQ files (R1 and R2) or single-end files.

Supported formats: `.fastq`, `.fq`, `.fastq.gz`, `.fq.gz`, `.bam`, `.vcf`

### 4. Run Analysis

Click **"Run Pipeline"** and configure:

- **Analysis Type**: WGS, WES, Gene Panel, or RNA-Seq
- **Run Mode**:
  - *Standard* — Balanced speed and accuracy
  - *Fast* — Reduced quality thresholds, skips BQSR (for limited resources)
  - *High Sensitivity* — Stricter quality thresholds, lower calling confidence
- **Resources**: Threads and memory allocation

### 5. Monitor Progress

Track real-time progress on the job detail page. View live logs under the **Log** tab.

### 6. Download Results

After completion, browse and download results from the **Results** tab:
- QC reports (FastQC, MultiQC HTML reports)
- Aligned BAM files with statistics
- Filtered and annotated VCF files
- Summary HTML report

## Pipeline Steps

### DNA Pipelines (WGS / WES / Gene Panel)

```
Raw FASTQ → QC (FastQC) → Trimming (fastp) → Alignment (BWA-MEM)
→ Deduplication (GATK/samtools) → BQSR (GATK, optional)
→ Variant Calling (GATK HaplotypeCaller / bcftools)
→ Variant Filtering → Annotation (SnpEff) → Report
```

### RNA-Seq Pipeline

```
Raw FASTQ → QC (FastQC) → Trimming (fastp) → Alignment (HISAT2/minimap2)
→ Quantification (featureCounts) → Report
```

## Configuration

### Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `SECRET_KEY` | auto-generated | JWT secret for authentication |
| `MAX_THREADS` | CPU count | Max threads per job |
| `MAX_MEMORY_GB` | 16 | Max memory per job (GB) |
| `MAX_CONCURRENT_JOBS` | 2 | Max parallel jobs |
| `UPLOAD_DIR` | `./data/uploads` | Upload storage path |
| `RESULTS_DIR` | `./data/results` | Results storage path |
| `REFERENCES_DIR` | `./data/references` | Reference genome path |

### Custom Tool Paths

Set environment variables to override tool locations:

```bash
export FASTQC_PATH=/custom/path/fastqc
export BWA_PATH=/custom/path/bwa
export GATK_PATH=/custom/path/gatk
# ... etc
```

## Troubleshooting

### Common Issues

**"Reference genome not found"**
Place your reference FASTA at `data/references/genome.fa`. See [Reference Genomes](#reference-genomes) section.

**"Max concurrent jobs reached"**
Wait for running jobs to finish, or increase `MAX_CONCURRENT_JOBS` in `docker-compose.yml`.

**Out of memory during alignment**
Reduce the threads and memory settings when starting a job, or increase the Docker container limits.

**Tool not found errors**
Check the System page to see which tools are installed. The pipeline will use fallback tools when possible (e.g., bcftools instead of GATK).
